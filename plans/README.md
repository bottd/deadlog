## Findings considered and rejected

- **Redundant "Load More" button alongside auto infinite-scroll** (`Changelog.svelte:186–202`): kept as-is — the button is a legitimate no-JS/observer-failure fallback and removing it risks UX regressions for marginal cleanup.
- **`EntityTooltip` naming / `ability→vitality`, `upgrade→spirit` color mapping**: likely deliberate domain mapping of item categories to color tokens; renaming without maintainer input risks churn. Recorded, not planned.
- **Empty/loading/error states**: audited and found already well-handled (skeletons, empty state, error+retry in `Changelog.svelte`) — no action.
- **Meta/OG/sitemap**: present and correct on all four routes — no action.
